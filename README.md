状態管理とUI分離を意識した、React × TypeScript Todoアプリ

シンプルで使いやすく、実務を意識した設計で作成した Todo アプリです。 状態管理・UI設計を重視し、ポートフォリオ用途として「読みやすさ」「拡張しやすさ」を重視して設計しています。


【制作目的】
⚫︎ React / TypeScript の基礎〜実務レベルの理解を定着させる
⚫︎ useReducer を用いた状態管理の整理
⚫︎ UI状態 / データ状態 / 派生状態の分離設計を実践
 「なぜそう設計するのか」を説明できるコードを書く

【URL】
https://todo-react-kiex.vercel.app/

【主な機能】
⚫︎ Todo の追加 / 削除 / 編集
⚫︎ 完了・未完了の切り替え
⚫︎ フィルター（すべて / 未完了 / 完了）
⚫︎ 並び替え（新しい順 / 古い順）
⚫︎ 残り件数の表示
⚫︎ 全削除（confirm 付き）
⚫︎ LocalStorage によるデータ永続化

【使用技術】
⚫︎ React
⚫︎ TypeScript
⚫︎ CSS（フレームワーク不使用）


【設計で工夫したポイント】
1. 状態を3種類に分けて管理
種類	内容	管理方法
データ状態	todos	useReducer
UI状態	filter / sortOrder	useState
派生状態	表示用リスト / 件数	計算で生成
 todos を直接加工せず、必ず派生状態を作ることで、 ロジックの破綻や予期しない副作用を防いでいます。

2. useReducer を採用
⚫︎ Todo の操作は「イベント駆動」で考えやすい
⚫︎ Action を通して状態変化が一元管理できる
⚫︎ コンポーネント分割後もロジックが散らばらない

3. UI とロジックの責務分離(コンポーネント分割)
⚫︎ TodoApp：状態管理・ロジック
⚫︎ Header（タイトル / 件数表示）
⚫︎ TodoInput（入力・追加）
⚫︎ FilterBar（フィルター / ソート）
⚫︎ TodoList / TodoItem（一覧表示）
⚫︎ DangerZone（全削除）

実務を想定し、読みやすさ・保守性を最優先しました。

4.UIとUX 
⚫︎ 20〜30代女性を想定した、やわらかく落ち着いた配色
⚫︎ 危険操作（削除）は目立ちすぎないが、迷わない配置

5. LocalStorage 
ページをリロードしても Todo が保持されるよう、useEffect を使って LocalStorage と同期しています。

useEffect(() => {
localStorage.setItem("todos", JSON.stringify(todos));
}, [todos]);
